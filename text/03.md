# 3 方法

针对复杂水域中的无人艇全覆盖路径规划问题，本文在 Glasius Bio-inspired Neural Network（GBNN）覆盖需求表达的基础上，提出一种基于滚动候选分支评价的在线规划方法。与传统 GBNN 方法在当前位置邻域内直接选择神经元活性值最大的下一元胞不同，本文方法在每个决策时刻由当前元胞向外展开有限深度候选分支，并从覆盖收益、运动连续性和死区风险等方面对候选分支进行分层筛选，最终仅执行最优分支的第一步。该策略既保留了 GBNN 计算量小、响应速度快的特点，又能够缓解单步贪心选点在开阔区域中易形成螺旋路径、在障碍边缘易发生重复覆盖和局部死区的问题。

## 3.1 水域栅格与神经元映射

全覆盖路径规划首先需要对连续水域环境进行离散化处理。本文将待覆盖水域表示为二维栅格地图，并将可探测区域划分为尺寸一致的元胞集合：

$$
\mathcal{C}=\{c_1,c_2,\ldots,c_N\}
$$

其中，每个元胞既是栅格地图中的一个基本空间单元，也是 GBNN 中的一个神经元节点。本文假设元胞边长等于无人艇的有效探测宽度，因此无人艇进入某一可通行元胞后，该元胞即可视为已被覆盖。通过这种栅格地图与神经网络的一一对应关系，二维水域环境中的障碍区、未覆盖区和已覆盖区可以统一映射为神经网络中的不同外部输入状态。

元胞状态定义为：

$$
s_i(k)=
\begin{cases}
-1, & c_i \text{ 为障碍元胞}\\
0, & c_i \text{ 为未覆盖可通行元胞}\\
1, & c_i \text{ 为已覆盖元胞}

\end{cases}
$$

其中，$k$ 表示决策时刻，无人艇当前位置记为 $p(k)$。当无人艇移动至元胞 $c_i$ 后，该元胞状态更新为：

$$
s_i(k+1)=1
$$

由此，全覆盖任务可转化为在可通行元胞集合上的逐元胞访问问题；障碍元胞在规划过程中始终作为不可通行区域处理。

## 3.2 GBNN 覆盖需求场构建

GBNN 可视为一种拓扑组织的神经动态模型，其网络结构由栅格地图直接确定。与传统神经网络不同，该模型不依赖训练过程，也不包含输入层、隐含层和输出层的监督学习结构，而是通过神经元活性值在局部邻域内的传播来反映环境中的覆盖需求。

在 GBNN 层中，每个元胞 $c_i$ 对应一个神经元。神经元外部输入由当前覆盖状态决定：

$$
I_i(k)=
\begin{cases}
1, & s_i(k)=0\\
0, & s_i(k)=1\\
-1, & s_i(k)=2
\end{cases}
$$

其中，未覆盖元胞提供正向兴奋输入，表示该区域仍具有覆盖需求；障碍元胞提供负向抑制输入，表示不可通行区域；已覆盖元胞外部输入为零，其活性主要由邻域传播项决定。

神经元之间采用 8 邻域侧向连接。第 $i$ 个神经元的活性值记为 $x_i(k)$，其离散更新形式为：

$$
x_i(k+1)=g\left(I_i(k)+\sum_{j\in\mathcal{N}_i}w_{ij}[x_j(k)]^+\right)
$$

其中，$\mathcal{N}_i$ 为第 $i$ 个神经元的 8 邻域，$w_{ij}$ 为邻接权重，$[x]^+=\max(x,0)$，$g(\cdot)$ 为限幅函数。该更新机制使未覆盖区域产生的正活性能够通过侧向连接向周围传播，而障碍区域产生的负活性不向外扩散，仅在局部起抑制作用。因此，未覆盖区域通常保持较高活性，障碍区域保持较低活性，已覆盖区域的活性则随邻域状态逐渐变化。

需要说明的是，本文中的 GBNN 不直接给出完整覆盖路径，而是用于形成随覆盖状态动态变化的覆盖需求场。无人艇的实际运动决策由后续滚动候选分支评价机制完成。

## 3.3 传统 GBNN 选点策略及其局限

在传统 GBNN/BINN 全覆盖路径规划中，无人艇通常在当前位置邻域内选择下一元胞。为兼顾覆盖需求和航向连续性，常用选点准则可写为：

$$
p(k+1)=
\arg\max_{c_j\in \mathcal{A}(p(k))}
\left(x_j(k)+c y_j(k)\right)
$$

其中，$\mathcal{A}(p(k))$ 为当前位置的可通行邻域，$x_j(k)$ 为候选元胞对应的神经元活性，$y_j(k)$ 为航向连续性项，$c$ 为调节系数。该策略通过神经元活性引导无人艇靠近未覆盖区域，并通过航向项抑制不必要的急转弯。

然而，上述方法本质上仍是单步局部决策。它只能判断当前邻域内哪个元胞活性更高、转向代价更小，而难以评估该动作对未来若干步覆盖形态的影响。当无人艇处于开阔区域时，单步最大活性选点容易诱发局部螺旋式推进；当无人艇靠近障碍物、边界或凹陷区域时，则可能出现重复覆盖、短距离折返以及进入死区后再寻找逃逸路径的问题。

因此，本文不再将“邻域最大神经元活性”作为唯一选点依据，而是将 GBNN 活性场作为覆盖需求信息输入，并在此基础上引入有限深度滚动候选分支评价。

## 3.4 滚动候选分支生成

为克服单步选点的短视性，本文在每个决策时刻构造候选分支树。设无人艇在时刻 $k$ 位于元胞 $p(k)$，从该元胞出发，在可通行邻域内向前展开深度为 $H$ 的候选分支。

一条候选分支表示为：

$$
\pi=\{p^0,p^1,\ldots,p^H\}
$$

其中：

$$
p^0=p(k)
$$

且每一步满足：

$$
p^{h+1}\in \mathcal{A}(p^h),\quad h=0,1,\ldots,H-1
$$

所有候选分支组成集合：

$$
\Pi_H(k)=\{\pi_1,\pi_2,\ldots,\pi_M\}
$$

候选分支树并非用于预先搜索全局最优覆盖路径，而是用于描述当前状态下未来若干步可能形成的局部覆盖结果。算法对每条候选分支进行评价，但最终只执行最优分支的第一步。进入下一决策时刻后，元胞状态、外部输入、神经元活性场以及候选分支集合均重新更新，从而形成滚动优化过程。

## 3.5 候选分支分层评价

为避免将评价函数设计为多权重简单叠加的形式，本文采用分层筛选式评价流程。候选分支评价包括合法性筛选、覆盖收益筛选和结构质量选择三个阶段。

首先，对候选分支进行合法性筛选。若分支满足以下情况之一，则剔除：

1. 经过障碍元胞；
2. 存在明显的立即往返；
3. 连续重复覆盖次数过多；
4. 转向过于频繁；
5. 分支末端具有较高死区风险。

筛选后的候选分支集合记为：

$$
\Pi_H^{valid}(k)
$$

其次，对合法分支计算覆盖收益。覆盖收益由新增覆盖数量和 GBNN 活性辅助项构成：

$$
G_{\mathrm{cov}}(\pi)
=
N_{\mathrm{new}}(\pi)
+
\lambda A_{\mathrm{gbnn}}(\pi)
$$

其中：

$$
N_{\mathrm{new}}(\pi)=
\sum_{h=1}^{H}
\mathbf{1}\left(s(p^h,k)=0\right)
$$

表示候选分支在未来 $H$ 步内能够新增覆盖的元胞数量；

$$
A_{\mathrm{gbnn}}(\pi)=
\frac{1}{H}\sum_{h=1}^{H}x_{p^h}(k)
$$

表示候选分支经过元胞的平均神经元活性。$\lambda$ 取较小值，使 GBNN 活性主要发挥覆盖需求引导作用，而不替代新增覆盖数量的主导地位。

记最大覆盖收益为：

$$
G_{\max}=
\max_{\pi\in\Pi_H^{valid}(k)}
G_{\mathrm{cov}}(\pi)
$$

保留覆盖收益接近最优的候选分支：

$$
\Pi_H^{top}(k)=
\left\{
\pi\in\Pi_H^{valid}(k)
\mid
G_{\mathrm{cov}}(\pi)\geq G_{\max}-\delta
\right\}
$$

其中，$\delta$ 为覆盖收益容忍阈值。该处理避免算法因微小收益差异而选择运动结构较差的分支。

最后，在 $\Pi_H^{top}(k)$ 中选择结构质量最优的分支。结构质量主要考虑航向连续性、转向次数、重复覆盖程度、是否沿已覆盖区域边界外侧推进，以及是否降低后续进入死区的风险。用结构代价 $C_{\mathrm{struct}}(\pi)$ 表示，则最优候选分支为：

$$
\pi^*(k)=
\arg\min_{\pi\in\Pi_H^{top}(k)}
C_{\mathrm{struct}}(\pi)
$$

该过程体现了“先保证分支可行，再保证覆盖收益，最后优化路径结构”的决策逻辑。与传统方法在单个邻域点上比较神经元活性不同，本文在局部路径层面对候选分支进行评价，因此能够提前识别可能导致折返、重复覆盖或死区的运动趋势。

## 3.6 滚动执行机制

虽然算法在每个决策时刻得到的是一条长度为 $H$ 的候选分支，但无人艇并不一次性执行完整分支，而是仅执行该分支的第一步：

$$
p(k+1)=p_*^1
$$

其中：

$$
\pi^*(k)=\{p(k),p_*^1,p_*^2,\ldots,p_*^H\}
$$

无人艇进入 $p(k+1)$ 后，执行以下更新：

1. 将当前元胞标记为已覆盖；
2. 更新元胞访问次数；
3. 更新覆盖率和重复覆盖率；
4. 根据元胞状态重新设置外部输入；
5. 更新 GBNN 神经元活性场；
6. 重新生成下一时刻候选分支树。

因此，该方法本质上是一种基于 GBNN 覆盖需求场的滚动优化过程。它不依赖离线生成的完整路径，而是在每一步结合当前覆盖状态和局部未来分支评价动态选择下一元胞，使规划过程能够持续适应已覆盖区域、障碍边界和局部死区的变化。

## 3.7 死区检测与逃逸点选择

当滚动候选分支无法继续有效推进时，算法进入死区逃逸阶段。触发条件包括：

1. $\Pi_H^{valid}(k)$ 为空；
2. 连续若干步覆盖率无明显提升；
3. 无人艇持续重复覆盖但无新增覆盖；
4. 当前区域附近不存在可继续覆盖的有效分支。

在传统 GBNN 方法中，未覆盖区域的正活性可以通过侧向连接传播至死区附近，使无人艇沿活性梯度逐渐离开死区。本文保留这一思想，但进一步强调逃逸点选择而非仅依赖局部活性爬升。具体地，本文采用回溯候选与 Dijkstra 扩张候选相结合的方式生成逃逸点。

回溯候选点沿历史路径反向查找，寻找最近一个仍具有继续覆盖潜力的历史元胞。该元胞应满足其邻域内存在未覆盖可通行元胞，或从该元胞重新展开候选分支后能够恢复正常覆盖。该方式适合处理凹陷区、狭窄通道和障碍夹角附近的死区。

Dijkstra 扩张候选点则从当前位置出发，在可通行元胞图上进行代价扩张搜索。与普通几何膨胀不同，该搜索只沿可通行元胞扩展，不能穿越障碍，因此能够避免选择“几何距离近但实际不可达或绕行代价很高”的逃逸点。搜索目标为可达的未覆盖元胞或未覆盖连通域入口。

两类候选点组成逃逸点集合：

$$
\mathcal{E}(k)=
\mathcal{E}_{back}(k)
\cup
\mathcal{E}_{dij}(k)
$$

最终选择连接代价最低且后续覆盖潜力较好的逃逸点：

$$
e^*(k)=
\arg\min_{e\in\mathcal{E}(k)}
C_{\mathrm{esc}}(p(k),e)
$$

其中，$C_{\mathrm{esc}}$ 可综合考虑路径长度、重复覆盖、转向代价和逃逸点周围未覆盖区域规模。确定逃逸点后，使用图搜索方法生成连接路径：

$$
P_{\mathrm{esc}}=
\mathrm{GraphSearch}(p(k),e^*)
$$

无人艇沿该连接路径移动至逃逸点。在连接过程中，若经过未覆盖元胞，也同步将其标记为已覆盖。到达逃逸点后，算法重新进入滚动候选分支评价阶段。

## 3.8 算法流程

完整流程如下。

```text
Algorithm: Rolling-Optimization GBNN Complete Coverage Planning
Input:
    Cell map C
    Initial position p(0)
    Obstacle cells
    Rolling depth H
    Target coverage rate
Initialize:
    Set obstacle cells as s_i = 2
    Set traversable cells as s_i = 0
    Set initial cell as covered
    Build the grid-neuron mapping
    Build the single-layer GBNN
    Initialize USV path and visit count
while coverage_rate < target_rate and step < max_steps:
    Update external input I_i according to cell state s_i
    Update GBNN activity field X
    Generate candidate branch set Pi_H(k) from current cell p(k)
    Remove invalid branches:
        branches crossing obstacles
        branches with immediate backtracking
        branches with excessive repetition
        branches with frequent heading changes
        branches with high trap risk
    if valid branches exist:
        Compute coverage gain G_cov for each valid branch
        Select near-optimal branches Pi_H^top(k)
        Choose the branch with the best structural quality
        Move to the first cell of the selected branch
        Mark the new cell as covered
    else:
        Generate backtracking escape candidate
        Generate Dijkstra expansion escape candidate
        Select escape target with minimum escape cost
        Connect to escape target using graph search
        Move along escape path
    Update path, visit count, coverage rate, and metrics
Output:
    Complete coverage path
    Coverage map
    GBNN activity field
    Escape records
    Evaluation metrics
```

## 3.9 方法特点

与传统 GBNN 方法相比，本文方法的主要区别在于路径决策层。传统 GBNN 利用未覆盖区域的兴奋输入和障碍区域的抑制输入形成神经元活性分布，并在每一步选择邻域内综合活性值最大的元胞。该方式计算简洁、在线响应能力强，但决策视野仅限于单步邻域。

本文方法仍采用 GBNN 描述覆盖需求场，但将实际移动决策扩展为有限深度候选分支评价。算法先通过合法性筛选排除不可行或高风险分支，再以新增覆盖数量和 GBNN 活性衡量覆盖收益，最后从接近最优收益的分支中选择运动结构更平滑、重复覆盖更少、死区风险更低的分支。最终执行第一步并滚动更新。

因此，本文方法既继承了 GBNN 计算量小、活性传播快、能够在线响应覆盖状态变化的优点，又通过局部未来分支评价改善了传统单步选点容易产生的螺旋推进、局部折返和死区逃逸效率不足等问题。
